# 卡牌职业与类别清单

## 当前能看出的卡牌规模

按当前导出统计，至少可以看到以下分类规模：

- `regent`：159
- `necrobinder`：140
- `defect`：114
- `colorless`：113
- `silent`：109
- `ironclad`：100
- `event`：36
- `curse`：23
- `token`：21
- `status`：18
- `quest`：5
- `root`：2

这里的统计是“导出后的卡面插画分类规模”，用于帮助理解资源覆盖范围，不等于完整玩法数据库，也不等于最终可玩卡牌总表。

## 按职业看这批资源

### Ironclad

当前可见数量为 100。这个体量说明 Ironclad 的卡面导出已经比较完整，足够研究：

- 起始牌与早期常见牌的视觉风格
- 攻击向卡牌的卡面倾向
- 红色职业框与插画之间的搭配关系

从已能看到的示例名看，这部分至少包含 `bash`、`break`、`anger` 等代表性条目。

### Silent

当前可见数量为 109。Silent 的导出规模同样不小，足够观察：

- 毒、匕首、潜行相关语义在卡面中的表达
- 绿色职业框与整体插画气质的统一方式
- 与无色牌之间的视觉边界

可见示例中包含 `corpse_explosion` 这类识别度很高的条目。

### Defect

当前可见数量为 114。Defect 的导出结果适合重点研究：

- 科技感、能量感与 orb 语义如何体现在卡面里
- 蓝色职业框与插画内容的结合方式
- power、skill、attack 的视觉差异是否明显

当前可见示例包括 `biased_cognition`、`quadcast` 等。

### Necrobinder

当前可见数量为 140，是当前非常值得重视的一类。它的规模说明这不是零星占位资源，而是一整套成熟职业内容。

从题材上推测，这类卡牌通常更适合观察：

- 死灵、召唤、契约、守卫类语义如何做插画表达
- 粉色系职业框和特殊风格卡图如何统一
- 该职业是否拥有更多 token、衍生物或体系牌支持

当前能看到的示例包括 `forbidden_grimoire`、`protector`。

### Regent

当前可见数量为 159，是目前导出规模最大的职业类资源之一。这说明 Regent 在当前发布版中的美术资源占比相当高，也意味着：

- 该职业可能拥有较丰富的卡池
- 职业视觉语义已经相当完整
- 做职业专题时很值得单独展开

当前可见示例包括 `meteor_shower`、`the_sealed_throne`。

## 非职业类卡牌也很重要

### Colorless

数量为 113。这个规模已经说明无色牌并不是边缘内容，而是一块相当独立、相当完整的资源集合。它对研究很有价值，因为它能帮助你区分：

- 哪些卡图真正受职业主题强约束
- 哪些卡图更偏通用效果、通用事件或跨职业语义

### Event

数量为 36。事件牌通常不以普通职业卡的逻辑出现，因此它们在画面叙事、边框语义和用途上都值得单独看。

### Curse / Status / Token / Quest

这几类虽然数量不如职业主卡池多，但对理解系统完整性非常重要：

- `curse` 说明负面牌也有独立视觉语义
- `status` 说明临时状态牌与系统干扰牌并非简单复用普通卡图
- `token` 说明部分体系可能依赖衍生物或中间对象
- `quest` 说明任务或章节相关牌型拥有自己的视觉分支

## 这份分类最有价值的地方

### 看出项目新增职业的投入程度

如果一个职业只出现几张卡，那更像测试残留或未完成内容；但当某职业已经达到 100 张以上时，就足以说明它在资源层已经是完整模块。

### 看出游戏不是只围绕三大基础职业扩展

当前导出里，`necrobinder` 和 `regent` 的体量很高，说明这个项目的职业层扩展远不止原版的几条主线。

### 看出卡牌系统并不只服务于“可打出的普通牌”

事件牌、任务牌、状态牌、token、诅咒牌都单独存在，说明卡牌 UI 这套系统承担了更广泛的叙事与机制职责。

## 如果要继续深入，下一步该怎么读

- 想研究“卡牌是怎么显示的”，下一步看 [卡牌渲染](/analysis/card-rendering)
- 想研究“为什么这些职业卡会长得不一样”，重点看职业框、横幅和材质分层
- 想做职业主题 Mod，可以先围绕单一职业做替换实验，再处理跨职业公共模板
