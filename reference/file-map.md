# 文件地图

## 这页怎么读

站点访客通常拿不到维护者本地的原始目录，所以这页不再写死本地路径，而是把研究材料按“用途”做成概念地图。你可以把它理解成：**这套站点背后到底依赖了哪些分析成果。**

## 研究材料总览

### 发布包样本

这是整个分析的起点。它负责回答：

- 游戏最终是以什么形式发布的
- 发布版里还能保留多少引擎与资源结构
- 哪些内容已经不可逆地丢失

### 恢复工程

这是最接近“发布态 Godot 工程视图”的材料，主要用来看：

- 场景结构
- 图集与材质
- 纹理与 UI 资源
- shader 与部分资源引用关系

### 结构化代码骨架

这是围绕 `sts2.dll` 整理出的类型与成员地图，主要用来看：

- 游戏有哪些核心模块
- 某个系统大致挂在哪类对象上
- 想改行为时应该往哪一层追

### 卡牌导出结果

这部分帮助回答：

- 卡牌插画资源是否足够做职业分类
- 卡牌规模是否已经覆盖多职业与多类别
- 卡牌显示更像图集切片还是独立图片

### Spine 汇总结果

这部分帮助回答：

- Spine 在项目里覆盖哪些内容
- 角色、怪物、背景、特效的分布比例如何
- 哪些资源更适合做兼容性实验和替换测试

### 模组样本与分析笔记

这部分更偏实践导向，主要用来看：

- 现有 Mod 大概如何组织
- 做资源替换和做 DLL 改动的边界在哪里
- 当前分析中有哪些高价值结论和已知限制

## 如何使用这张地图

- 想理解项目整体：优先看恢复工程 + 结构化代码骨架
- 想研究卡牌：优先看卡牌导出结果 + 卡牌渲染专题
- 想研究动画：优先看 Spine 汇总结果 + 版本兼容专题
- 想做 Mod：优先看模组样本 + Mod 制作章节
